# 合同审查系统 - 接口说明文档

## 概述

本文档描述了合同审查系统中与合同预览相关的两个核心接口。

---

## 1. 获取合同预览信息接口

### 基本信息

| 属性 | 说明 |
|------|------|
| **接口地址** | `GET /api/contract/preview/{contractId}` |
| **功能描述** | 获取指定合同的预览元数据信息，用于前端展示合同预览对话框 |
| **Controller** | `ContractController.java` (第71-76行) |
| **Service实现** | `ContractServiceImpl.java` (第250-283行) |

### 请求参数

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| Authorization | String | Header | 是 | 用户认证Token |
| contractId | Long | Path | 是 | 合同ID |

### 响应数据 (ContractPreviewVO)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| contractId | Long | 合同ID |
| fileName | String | 文件名 |
| fileType | String | 文件类型 (pdf/docx/doc) |
| fileUrl | String | 文件访问URL (用于调用 `/contract/file/{id}` 接口) |
| fileSize | Long | 文件大小（字节） |

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "contractId": 2,
    "fileName": "智能合同审查助手 - 知识库文档+简历项目描述.docx",
    "fileType": "docx",
    "fileUrl": "/contract/file/2",
    "fileSize": 102400
  }
}
```

### 业务流程

1. 验证用户Token，获取用户ID
2. 查询合同是否存在
3. 校验用户是否有权查看该合同（只能查看自己的合同）
4. 构建预览VO对象返回

### 错误码

| 错误码 | 说明 |
|--------|------|
| 401 | 用户认证信息无效 |
| 404 | 合同不存在 |
| 403 | 无权操作该合同 |

## 2. 获取合同文件接口

### 基本信息

| 属性 | 说明 |
|------|------|
| **接口地址** | `GET /api/contract/file/{contractId}` |
| **功能描述** | 获取合同文件的二进制流，用于在线预览或下载合同文件 |
| **Controller** | `ContractController.java` (第79-85行) |
| **Service实现** | `ContractServiceImpl.java` (第287-325行) |

### 请求参数

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| Authorization | String | Header | 是 | 用户认证Token |
| contractId | Long | Path | 是 | 合同ID |

### 响应数据

| 属性 | 说明 |
|------|------|
| Content-Type | 根据文件类型返回对应的MIME类型 |
| Content-Disposition | `inline; filename="xxx"`（浏览器内联展示） |
| Body | 文件二进制流 |

### 支持文件类型

| 文件类型 | MIME类型 |
|----------|----------|
| pdf | `application/pdf` |
| docx | `application/vnd.openxmlformats-officedocument.wordprocessingml.document` |
| doc | `application/msword` |
| 其他 | `application/octet-stream` |

### 业务流程

1. 验证用户Token，获取用户ID
2. 查询合同是否存在
3. 校验用户是否有权访问该合同
4. 根据文件路径读取文件资源
5. 设置响应头（Content-Type、Content-Disposition）
6. 返回文件流

### 错误码

| HTTP状态码 | 说明 |
|------------|------|
| 401 | 用户认证信息无效 |
| 404 | 合同不存在或文件不存在/无法读取 |
| 403 | 无权访问该合同 |

## 接口协作流程

```
┌─────────┐     GET /api/contract/preview/2      ┌─────────┐
│         │ ───────────────────────────────────> │         │
│   前端   │                                      │   后端   │
│         │ <─────────────────────────────────── │         │
└─────────┘     返回 ContractPreviewVO            └─────────┘
      │
      │ 提取 fileUrl
      ▼
┌─────────┐     GET /api/contract/file/2         ┌─────────┐
│         │ ───────────────────────────────────> │         │
│   前端   │                                      │   后端   │
│         │ <─────────────────────────────────── │         │
└─────────┘     返回文件二进制流                   └─────────┘
      │
      ▼
┌─────────┐
│ 文件预览  │
│ 对话框展示 │
└─────────┘
```

---

## 典型使用场景

1. 用户在合同列表页点击"预览"按钮
2. 前端先调用 `/preview/{id}` 获取文件元数据（文件名、类型、大小）
3. 前端根据返回的 `fileUrl` 调用 `/file/{id}` 获取实际文件流
4. 前端使用 `FilePreviewDialog.vue` 组件展示文件内容

---

## 前端调用示例

```javascript
// 1. 获取预览信息
const res = await getContractPreview(contractId);
if (res.code === 200 && res.data) {
  const { fileName, fileType } = res.data;
  const fileUrl = getContractFileUrl(contractId);
  
  // 2. 打开预览对话框
  previewDialogRef.value?.open(fileUrl, fileName, fileType);
}
```

---

## 安全说明

- 两个接口都需要携带有效的 `Authorization` Token
- 用户只能访问自己上传的合同文件
- 系统会校验合同的所有权，防止越权访问
- 文件路径存储在数据库中，通过合同ID映射获取，避免目录遍历攻击
