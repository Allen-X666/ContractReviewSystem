# 对话中断机制使用说明

## 概述

系统支持用户随时中断 AI 的思考和输出。当用户点击"中断"按钮时，后台的 AI 处理会立即停止，不再返回任何内容。

## 后端实现

### 1. 中断管理器

文件：`合同审查/app/utils/conversation_stopper.py`

- 使用内存存储中断状态
- 支持标记对话为中断状态
- 支持检查对话是否被中断
- 自动清理过期的中断标记（5分钟）

### 2. API 接口

#### 中断对话

```http
POST /api/v1/chat-bot/conversations/{conversation_id}/stop
```

**请求参数：**
- `conversation_id`: 对话ID（路径参数）

**响应示例：**
```json
{
  "code": 200,
  "message": "对话已中断",
  "conversation_id": "xxx-xxx-xxx"
}
```

### 3. SSE 输出格式

正常完成时：
```
data: {"content": "AI 的回复内容"}

data: [DONE]

```

被中断时：
```
data: [STOPPED]

```

## 前端集成指南

### 1. 添加中断按钮

在对话界面添加一个中断按钮（如用户选中的按钮）：

```html
<button 
  class="el-button el-button--danger send-btn"
  @click="stopConversation"
  v-if="isGenerating"
>
  <i class="el-icon">
    <!-- 停止图标 -->
    <svg>...</svg>
  </i>
</button>
```

### 2. 实现中断功能

```javascript
// Vue 示例
export default {
  data() {
    return {
      conversationId: null,
      isGenerating: false,
      eventSource: null,
    };
  },
  
  methods: {
    // 发送消息
    async sendMessage(content) {
      this.isGenerating = true;
      
      // 创建 EventSource 连接
      const response = await fetch('/api/v1/chat-bot/messages', {
        method: 'POST',
        headers: {
          'Content-Type': 'text/plain',
          'Authorization': `Bearer ${this.token}`,
        },
        body: content,
      });
      
      // 使用 ReadableStream 读取 SSE
      const reader = response.body.getReader();
      const decoder = new TextDecoder();
      
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        
        const chunk = decoder.decode(value);
        const lines = chunk.split('\n');
        
        for (const line of lines) {
          if (line.startsWith('data: ')) {
            const data = line.slice(6);
            
            if (data === '[DONE]') {
              // 正常完成
              this.isGenerating = false;
              return;
            } else if (data === '[STOPPED]') {
              // 被中断
              this.isGenerating = false;
              this.$message.info('对话已中断');
              return;
            } else {
              // 正常内容
              try {
                const parsed = JSON.parse(data);
                this.appendContent(parsed.content);
              } catch (e) {
                console.error('解析错误:', e);
              }
            }
          }
        }
      }
    },
    
    // 中断对话
    async stopConversation() {
      if (!this.conversationId || !this.isGenerating) return;
      
      try {
        const response = await fetch(
          `/api/v1/chat-bot/conversations/${this.conversationId}/stop`,
          {
            method: 'POST',
            headers: {
              'Authorization': `Bearer ${this.token}`,
            },
          }
        );
        
        const result = await response.json();
        if (result.code === 200) {
          this.$message.success('已中断对话');
          this.isGenerating = false;
        } else {
          this.$message.error('中断失败');
        }
      } catch (error) {
        console.error('中断对话失败:', error);
        this.$message.error('中断失败');
      }
    },
  },
};
```

### 3. 状态管理

- `isGenerating`: 控制中断按钮的显示/隐藏
- 当 AI 开始生成回复时设为 `true`
- 当收到 `[DONE]` 或 `[STOPPED]` 时设为 `false`

### 4. 用户体验优化

1. **按钮状态**
   - 发送消息时显示中断按钮
   - 中断后恢复发送按钮

2. **视觉反馈**
   - 中断后立即停止显示"正在输入..."
   - 可以显示提示"对话已中断"

3. **错误处理**
   - 网络错误时给出友好提示
   - 中断失败时允许重试

## 注意事项

1. **中断时机**
   - 中断只影响当前正在生成的回复
   - 已经生成的内容会保留
   - 用户可以继续发送新消息

2. **并发处理**
   - 每个对话独立管理中断状态
   - 中断一个对话不会影响其他对话

3. **过期清理**
   - 中断标记会在5分钟后自动过期
   - 新消息会自动清除之前的中断标记

## 测试示例

```bash
# 1. 发送消息（在另一个终端执行）
curl -X POST http://localhost:8001/api/v1/chat-bot/messages \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: text/plain" \
  -d "帮我分析合同"

# 2. 立即中断对话（在第三个终端执行）
curl -X POST http://localhost:8001/api/v1/chat-bot/conversations/YOUR_CONVERSATION_ID/stop \
  -H "Authorization: Bearer YOUR_TOKEN"
```
