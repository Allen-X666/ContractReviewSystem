# 智能客服前端设计方案

## 一、需求概述

在现有合同审查系统中新增"智能客服"功能，以浮动按钮+弹窗的形式呈现，用户可在任意页面右下角唤起客服进行对话交互。

### 核心需求
| 序号 | 需求 | 说明 |
|------|------|------|
| 1 | 浮动按钮 | 固定在页面右下角，支持长按拖拽移动，刷新后恢复默认位置 |
| 2 | 对话弹窗 | 点击按钮展开聊天界面，支持发送消息、AI 回复 |
| 3 | 历史对话管理 | 查看历史对话列表、删除、重命名对话 |
| 4 | 导出 Excel | 将对话记录导出为 Excel 文件 |

---

## 二、技术方案

### 2.1 技术栈

| 技术 | 用途 |
|------|------|
| Vue 3 Composition API | 组件开发 |
| Pinia Store | 对话状态管理与持久化 |
| Element Plus | UI 组件库（弹窗、表单、表格、消息提示） |
| localStorage | 对话历史持久化存储 |
| 手动 Blob + URL | 导出 Excel（无需额外依赖） |

### 2.2 文件结构

```
frontend/src/
├── components/
│   └── ChatBot/
│       ├── ChatBot.vue              # 主组件（浮动按钮 + 弹窗容器）
│       ├── ChatPanel.vue            # 聊天面板（消息发送/接收）
│       └── ChatHistory.vue          # 历史对话管理面板
├── stores/
│   └── chatbot.js                   # Pinia store（对话状态管理）
```

### 2.3 页面布局设计

#### 浮动按钮
- 位置：固定定位 `position: fixed`，右下角（默认 bottom: 32px, right: 32px）
- 样式：圆形 56px，主题色渐变背景，带客服图标
- 交互：长按 300ms 触发拖拽模式，鼠标/触摸跟随移动
- 持久化：拖拽结束位置保存到 localStorage，页面刷新后恢复

#### 对话弹窗
- 宽度：400px，高度：560px
- 位置：在浮动按钮上方弹出（如按钮被拖拽，弹窗跟随）
- 结构：
  - 顶部：标题栏 + 功能按钮（历史、新建、关闭）
  - 中部：消息列表区域（支持滚动，气泡样式）
  - 底部：输入框 + 发送按钮

#### 历史对话面板
- 在弹窗内切换为历史列表视图
- 每条对话记录显示：名称、最后消息预览、时间
- 操作：查看、重命名、删除
- 顶部：导出 Excel 按钮 + 新建对话按钮

## 三、交互流程

### 3.1 发送消息流程
1. 用户在输入框输入消息 → 点击发送
2. 消息追加到当前对话的 messages 数组
3. 显示"正在思考..."的打字动画
4. 模拟 AI 回复（前端本地模拟，不依赖后端）
5. 消息持久化到 localStorage

### 3.2 历史对话管理流程
1. 点击历史按钮 → 切换到历史列表视图
2. 列表显示所有历史对话，按最后更新时间倒序
3. 点击对话项 → 切换回聊天视图并加载该对话
4. 长按/右键菜单 → 重命名、删除操作
5. 点击导出 → 当前对话导出为 Excel

### 3.3 拖拽交互流程
1. 鼠标在按钮上按下并保持 300ms → 进入拖拽模式
2. 鼠标移动时，按钮跟随移动（限制在视口范围内）
3. 鼠标释放 → 退出拖拽模式，保存位置到 localStorage
4. 普通点击（非拖拽）→ 打开/关闭对话弹窗

## 四、数据模型

### 4.1 对话数据结构

```js
{
  id: 'chat_1700000000000',        // 对话ID（时间戳）
  name: '新对话',                    // 对话名称
  messages: [
    {
      id: 'msg_1700000000001',
      role: 'user',                  // 'user' | 'assistant' | 'system'
      content: '你好，请问如何审查合同？',
      timestamp: 1700000000001
    },
    {
      id: 'msg_1700000000002',
      role: 'assistant',
      content: '您好！我是智能客服助手...',
      timestamp: 1700000000002
    }
  ],
  createdAt: 1700000000000,
  updatedAt: 1700000000002
}
```

### 4.2 按钮位置数据

```js
{
  x: null,  // 使用默认位置（右下角）
  y: null   // 从 localStorage 读取
}
```

## 五、样式规范

遵循项目现有主题变量（`variables.scss`）：

| 元素 | 样式 |
|------|------|
| 浮动按钮背景 | `linear-gradient(135deg, $primary-color, $primary-light)` |
| 用户消息气泡 | `background: $primary-color; color: white` |
| AI 消息气泡 | `background: $bg-tertiary; color: $text-primary` |
| 弹窗阴影 | `box-shadow: $shadow-xl` |
| 圆角 | `$radius-lg`（弹窗）、`50%`（浮动按钮） |
| 过渡动画 | `$transition-base`（250ms ease） |
| 字体 | 项目统一 `$font-family` |

## 六、AI 回复策略

由于仅修改前端代码，AI 回复采用本地模拟方案：
- 关键词匹配：根据用户输入的关键词返回预设的合同审查相关回复
- 默认回复：对未知问题返回通用引导信息
- 打字机效果：逐字显示回复内容，增强交互感
