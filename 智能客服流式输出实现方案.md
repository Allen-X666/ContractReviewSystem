# 智能客服流式输出实现方案

## 一、架构概览

```
┌─────────────┐    SSE / EventSource    ┌──────────────┐   HTTP Streaming    ┌──────────────┐   DashScope API   ┌──────────────┐
│   Vue3 前端  │ <─────────────────────> │  Spring Boot │ <─────────────────> │   FastAPI    │ <───────────────> │  通义千问 LLM  │
│             │   逐 token 接收展示      │   (Java)     │   逐 chunk 转发     │  (Python)    │   流式调用         │  (DashScope) │
└─────────────┘                         └──────────────┘                     └──────────────┘                   └──────────────┘
```

**核心思路**：FastAPI 通过 DashScope SDK 的流式接口逐 token 获取 LLM 输出，以 SSE（Server-Sent Events）格式返回；Spring Boot 通过 `StreamingResponseBody` 逐 chunk 读取并透传给前端；前端使用 `EventSource` 或 `fetch + ReadableStream` 接收并实时渲染。

***

## 二、两种实现方案对比

| 维度       | 方案一：SSE 流式透传（推荐）        | 方案二：WebSocket   |
| -------- | ----------------------- | --------------- |
| 协议       | SSE（Server-Sent Events） | WebSocket       |
| 方向       | 单向：服务端 → 客户端            | 双向              |
| 连接方式     | 基于 HTTP，天然兼容            | 需独立升级协议         |
| Nginx 兼容 | 只需关闭 `proxy_buffering`  | 需配置 `Upgrade` 头 |
| 浏览器支持    | `EventSource` 原生 API    | 需第三方库或原生 API    |
| 实现复杂度    | 低                       | 中               |
| 适用场景     | AI 对话流式输出（单向推送）         | 需要双向通信的场景       |
| 与现有架构兼容  | 高（已有 SSE 进度推送先例）        | 需额外配置           |

**推荐方案一（SSE 流式透传）**，原因：与项目现有 SSE 推送架构一致，Nginx 已有相关配置基础，实现复杂度低。

## 三、方案一详细实现：SSE 流式透传

### 3.1 FastAPI 层

#### 3.1.1 新增 chatbot 路由模块

**文件路径**：`合同审查/app/api/v1/endpoints/chatbot.py`

FastAPI 层的核心职责是调用 DashScope 的流式 API，将 LLM 逐 token 输出以 SSE 格式返回。

```python
"""
智能客服 API 路由
"""
import json
import logging
from typing import Optional

from fastapi import APIRouter, HTTPException, status
from fastapi.responses import StreamingResponse

from 合同审查.app.llm.tongyi_llm import TongyiLLM, LLMConfig
from 合同审查.app.core.config import settings

logger = logging.getLogger(__name__)
router = APIRouter()


@router.post("/send")
async def send_message(content: str):
    """
    发送消息（SSE 流式输出）

    以 SSE 格式逐 token 返回 AI 回答，前端可实时渲染。

    请求体：纯文本字符串（用户消息）
    响应：text/event-stream 流
    """
    if not content or not content.strip():
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="消息内容不能为空"
        )

    config = LLMConfig(
        model=settings.LLM_MODEL_DEFAULT,
        temperature=0.7,
        max_tokens=2048,
        top_p=0.9,
        streaming=True,
    )
    llm = TongyiLLM(config)

    async def event_generator():
        try:
            from langchain_core.messages import HumanMessage, SystemMessage

            messages = [
                SystemMessage(content="你是一个专业的合同法律顾问智能客服，请用中文回答用户的问题。"),
                HumanMessage(content=content),
            ]

            for chunk in llm._stream(messages):
                token = chunk.message.content
                if token:
                    sse_data = json.dumps({"content": token}, ensure_ascii=False)
                    yield f"data: {sse_data}\n\n"

            yield "data: [DONE]\n\n"

        except Exception as e:
            logger.error(f"流式生成失败: {e}", exc_info=True)
            error_data = json.dumps({"error": str(e)}, ensure_ascii=False)
            yield f"data: {error_data}\n\n"

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
            "X-Accel-Buffering": "no",
        },
    )
```

#### 3.1.3 SSE 数据格式说明

每条 SSE 消息格式如下：

```
data: {"content": "你"}\n\n
data: {"content": "好"}\n\n
data: {"content": "，"}\n\n
...
data: [DONE]\n\n
```

- `data: `  是 SSE 协议前缀
- `\n\n` 是 SSE 消息分隔符（双换行）
- `[DONE]` 标记流式输出结束

### 3.2 Spring Boot 层

#### 3.2.1 Controller 层改造

**文件路径**：`ContractReview/src/main/java/com/example/contractreview/controller/ChatBotController.java`

Spring Boot 层的核心是将 FastAPI 返回的 SSE 流**原样透传**给前端，使用 `StreamingResponseBody` 实现。

```java
package com.example.contractreview.controller;

import com.example.contractreview.model.dto.ChatbotCreateDTO;
import com.example.contractreview.service.ChatBotService;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.servlet.mvc.method.annotation.StreamingResponseBody;

@RestController
@Slf4j
@Tag(name = "智能客服")
@RequestMapping("/chatbot")
@RequiredArgsConstructor
public class ChatBotController {

    private final ChatBotService chatBotService;

    @PostMapping("/messages")
    @Operation(summary = "发送消息（SSE 流式输出）")
    public ResponseEntity<StreamingResponseBody> sendMessage(
            @RequestHeader("Authorization") String authorization,
            @RequestBody ChatbotCreateDTO chatbotCreateDTO) {

        log.info("收到智能客服请求, conversationId: {}", chatbotCreateDTO.getConversationId());

        StreamingResponseBody stream = chatBotService.streamMessage(chatbotCreateDTO);

        return ResponseEntity.ok()
                .contentType(MediaType.TEXT_EVENT_STREAM)
                .header("Cache-Control", "no-cache")
                .header("Connection", "keep-alive")
                .header("X-Accel-Buffering", "no")
                .body(stream);
    }
}
```

```java
package com.example.contractreview.service.serviceImpl;

import com.example.contractreview.client.ChatBotClient;
import com.example.contractreview.model.dto.ChatbotCreateDTO;
import com.example.contractreview.service.ChatBotService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.web.servlet.mvc.method.annotation.StreamingResponseBody;

import java.io.OutputStream;
import java.nio.charset.StandardCharsets;

@Service
@Slf4j
@RequiredArgsConstructor
public class ChatBotServiceImpl implements ChatBotService {

    private final ChatBotClient chatBotClient;

    @Override
    public StreamingResponseBody streamMessage(ChatbotCreateDTO chatbotCreateDTO) {
        String content = chatbotCreateDTO.getContent();

        return (OutputStream outputStream) -> {
            try {
                chatBotClient.streamMessage(content, outputStream);
            } catch (Exception e) {
                log.error("流式消息处理失败: {}", e.getMessage(), e);
                String errorSse = "data: {\"error\": \"" + e.getMessage() + "\"}\n\n";
                outputStream.write(errorSse.getBytes(StandardCharsets.UTF_8));
                outputStream.flush();
            }
        };
    }
}
```

#### 3.2.3 ChatBotClient 改造

**文件路径**：`ContractReview/src/main/java/com/example/contractreview/client/ChatBotClient.java`

关键点：使用 `RestClient` 以**流式方式**读取 FastAPI 的响应体，逐 chunk 写入到前端的 `OutputStream`。

```java
package com.example.contractreview.client;

import com.example.contractreview.config.FastApiConfig;
import com.fasterxml.jackson.databind.ObjectMapper;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import org.springframework.http.client.SimpleClientHttpRequestFactory;
import org.springframework.stereotype.Component;
import org.springframework.web.client.RestClient;

import java.io.BufferedReader;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.io.OutputStream;
import java.nio.charset.StandardCharsets;

@Slf4j
@Component
public class ChatBotClient {

    private final RestClient restClient;
    private final FastApiConfig fastApiConfig;
    private final ObjectMapper objectMapper;

    public ChatBotClient(FastApiConfig fastApiConfig, ObjectMapper objectMapper) {
        this.fastApiConfig = fastApiConfig;
        this.objectMapper = objectMapper;

        SimpleClientHttpRequestFactory factory = new SimpleClientHttpRequestFactory();
        factory.setConnectTimeout(fastApiConfig.getConnectTimeout());
        factory.setReadTimeout(0);

        this.restClient = RestClient.builder()
                .requestFactory(factory)
                .baseUrl(fastApiConfig.getFullApiUrl())
                .defaultHeader(HttpHeaders.CONTENT_TYPE, MediaType.TEXT_PLAIN_VALUE)
                .defaultHeader(HttpHeaders.ACCEPT, MediaType.TEXT_EVENT_STREAM_VALUE)
                .build();
    }

    /**
     * 流式发送消息：读取 FastAPI 的 SSE 响应并逐行写入输出流
     */
    public void streamMessage(String content, OutputStream outputStream) {
        log.info("调用 FastAPI 流式发送消息, content: {}", content);

        try (InputStream responseStream = restClient.post()
                .uri("/chatbot/send")
                .contentType(MediaType.TEXT_PLAIN)
                .body(content)
                .retrieve()
                .body(InputStream.class)) {

            if (responseStream == null) {
                log.error("FastAPI 返回空响应流");
                String errorSse = "data: {\"error\": \"AI 服务返回空响应\"}\n\n";
                outputStream.write(errorSse.getBytes(StandardCharsets.UTF_8));
                outputStream.flush();
                return;
            }

            BufferedReader reader = new BufferedReader(
                    new InputStreamReader(responseStream, StandardCharsets.UTF_8));
            String line;
            while ((line = reader.readLine()) != null) {
                outputStream.write((line + "\n").getBytes(StandardCharsets.UTF_8));
                if (line.isEmpty()) {
                    outputStream.flush();
                }
            }
            outputStream.flush();

        } catch (Exception e) {
            log.error("流式调用 FastAPI 失败: {}", e.getMessage(), e);
            throw new RuntimeException("AI 服务流式调用失败: " + e.getMessage());
        }
    }
}
```

**核心说明**：

- `factory.setReadTimeout(0)` —— 设置读取超时为无限，因为流式输出的持续时间不确定
- 使用 `body(InputStream.class)` 获取原始输入流，逐行读取并写入前端输出流
- 当读到空行时执行 `flush()`，确保每个 SSE 消息及时推送到前端

### 3.3 Vue3 前端层

#### 3.3.1 API 封装

**文件路径**：`src/api/chatbot.js`

由于 `EventSource` 只支持 GET 请求，而发送消息是 POST，因此使用 `fetch` + `ReadableStream` 来接收 SSE 流。

```javascript
/**
 * 流式发送消息（SSE）
 *
 * 使用 fetch + ReadableStream 接收 SSE 流式响应。
 * EventSource 仅支持 GET，而发送消息需 POST，故采用此方案。
 *
 * @param {Object} params - 消息参数
 * @param {number} params.conversationId - 对话ID
 * @param {string} params.content - 消息内容
 * @param {Function} params.onToken - 每收到一个 token 时的回调
 * @param {Function} params.onDone - 流式输出完成时的回调
 * @param {Function} params.onError - 发生错误时的回调
 * @returns {AbortController} 用于取消请求的控制器
 */
export function sendMessageStream({ conversationId, content, onToken, onDone, onError }) {
  const controller = new AbortController()

  const token = localStorage.getItem('token') || ''

  fetch(`${import.meta.env.VITE_API_BASE_URL}/chatbot/messages`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
      'Accept': 'text/event-stream',
    },
    body: JSON.stringify({ conversationId, content }),
    signal: controller.signal,
  })
    .then(async (response) => {
      if (!response.ok) {
        const errorText = await response.text()
        onError?.(new Error(`HTTP ${response.status}: ${errorText}`))
        return
      }

      const reader = response.body.getReader()
      const decoder = new TextDecoder('utf-8')
      let buffer = ''

      while (true) {
        const { done, value } = await reader.read()
        if (done) break

        buffer += decoder.decode(value, { stream: true })

        const lines = buffer.split('\n')
        buffer = lines.pop()

        for (const line of lines) {
          if (line.startsWith('data: ')) {
            const data = line.slice(6).trim()

            if (data === '[DONE]') {
              onDone?.()
              return
            }

            try {
              const parsed = JSON.parse(data)
              if (parsed.error) {
                onError?.(new Error(parsed.error))
                return
              }
              if (parsed.content) {
                onToken(parsed.content)
              }
            } catch {
              // 非 JSON 数据，忽略
            }
          }
        }
      }

      onDone?.()
    })
    .catch((err) => {
      if (err.name !== 'AbortError') {
        onError?.(err)
      }
    })

  return controller
}
```

```vue
      loading.value = false
    },
  })

  loading.value = false
}
```

### 3.4 Nginx 配置

**文件路径**：`nginx-1.28.0/conf/nginx.conf`

需新增 SSE 流式代理配置，关键点是**关闭 proxy\_buffering**：

```nginx
# 智能客服 SSE 流式输出代理
location /api/chatbot/ {
    proxy_pass http://127.0.0.1:8080/api/chatbot/;
    proxy_http_version 1.1;
    proxy_set_header Connection '';
    proxy_buffering off;
    proxy_cache off;
    chunked_transfer_encoding on;

    proxy_read_timeout 300s;
    proxy_send_timeout 300s;

    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

**配置说明**：

- `proxy_buffering off` —— 最关键配置，禁止 Nginx 缓冲后端响应，确保每个 chunk 立即推送到客户端
- `proxy_cache off` —— 禁用缓存，避免 SSE 流被缓存
- `Connection ''` —— 清除 `Connection` 头，保持长连接
- `proxy_read_timeout 300s` —— 流式输出可能持续较长时间，设置较长超时

## 四、数据流完整链路

```
1. 用户在前端输入消息，点击发送
       │
       ▼
2. 前端 fetch POST /api/chatbot/messages
       │
       ▼
3. Nginx 反向代理转发到 Spring Boot 8080 端口
   （proxy_buffering off，不缓冲）
       │
       ▼
4. ChatBotController 接收请求，返回 StreamingResponseBody
   Content-Type: text/event-stream
       │
       ▼
5. ChatBotClient 以流式方式调用 FastAPI
   POST /api/v1/chatbot/send
   Accept: text/event-stream
   readTimeout=0（无限等待）
       │
       ▼
6. FastAPI event_generator 逐 chunk 调用 TongyiLLM._stream()
   DashScope 流式 API → 逐 token 输出
       │
       ▼
7. FastAPI 将每个 token 包装为 SSE 格式返回
   data: {"content": "你"}\n\n
   data: {"content": "好"}\n\n
       │
       ▼
8. ChatBotClient BufferedReader 逐行读取
   写入到 Spring Boot 的 OutputStream
       │
       ▼
9. StreamingResponseBody 将数据推送到 Nginx
   （Nginx 不缓冲，直接转发）
       │
       ▼
10. 前端 fetch ReadableStream 逐 chunk 接收
    解析 SSE 数据，调用 onToken 回调
    实时更新 aiReply 模板渲染
```

## 五、关键注意事项

### 5.1 超时配置

| 组件                     | 参数                   | 推荐值   | 说明                          |
| ---------------------- | -------------------- | ----- | --------------------------- |
| Nginx                  | `proxy_read_timeout` | 300s  | 流式输出可能持续数分钟                 |
| Spring Boot RestClient | `readTimeout`        | 0（无限） | 等待 FastAPI 流完成              |
| Spring Boot SseEmitter | 不适用                  | —     | 使用 StreamingResponseBody 替代 |
| FastAPI                | LLM 调用超时             | 120s  | DashScope 单次调用超时            |
| 前端 fetch               | 无内置超时                | —     | 通过 AbortController 手动取消     |

### 5.2 与现有审查进度 SSE 的区别

| 维度             | 审查进度推送                                 | 智能客服流式输出                                |
| -------------- | -------------------------------------- | --------------------------------------- |
| 模型             | Spring Boot 轮询 FastAPI + SseEmitter 推送 | Spring Boot 透传 FastAPI 的 SSE 流          |
| FastAPI 端      | 普通 HTTP 接口，返回当前进度                      | SSE StreamingResponse，逐 token 输出        |
| 推送频率           | 每 2 秒轮询一次                              | 实时逐 token                               |
| Spring Boot 实现 | SseEmitter + ScheduledExecutorService  | StreamingResponseBody + RestClient 流式读取 |
| 前端接收           | EventSource（GET）                       | fetch + ReadableStream（POST）            |

### 5.3 连接异常处理

- **前端取消请求**：使用 `AbortController.abort()` 中断 fetch 连接
- **FastAPI 异常**：在 `event_generator` 的 `except` 中输出错误 SSE 消息
- **Spring Boot 异常**：在 `StreamingResponseBody` 的 `catch` 中输出错误 SSE 消息
- **Nginx 超时**：设置合理的 `proxy_read_timeout`，过长浪费资源，过短导致流中断

### 5.4 多轮对话支持

当前方案为单轮流式输出。如需多轮对话，需要：

1. FastAPI 端维护对话历史（可通过 Redis 或内存存储 `conversationId` 对应的消息列表）
2. 每次流式请求时，将历史消息作为上下文传入 LLM
3. 流式输出完成后，将本轮对话存入历史

### 5.5 跨域与认证

- SSE 请求会携带 `Authorization` 头（因为使用 fetch 而非 EventSource）
- Nginx 已配置 `proxy_set_header` 转发认证信息
- CORS 配置已在 FastAPI 和 Spring Boot 中统一处理

### 7.2 打字机效果优化

可在前端加入简单的防抖逻辑，避免 DOM 频繁更新导致性能问题：

```javascript
let tokenBuffer = ''
let renderTimer = null

function onToken(token) {
  tokenBuffer += token
  if (!renderTimer) {
    renderTimer = requestAnimationFrame(() => {
      aiReply.value += tokenBuffer
      tokenBuffer = ''
      renderTimer = null
    })
  }
}
```

### 7.3 错误重试机制

可在前端封装重试逻辑，当流式连接异常中断时自动重试：

```javascript
async function sendWithRetry(params, maxRetries = 2) {
  for (let i = 0; i <= maxRetries; i++) {
    try {
      return await sendMessageStream(params)
    } catch (err) {
      if (i === maxRetries) throw err
      await new Promise(r => setTimeout(r, 1000 * (i + 1)))
    }
  }
}
```
